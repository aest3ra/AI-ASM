# ai-asm Benchmark Harness

`bench/run_bench.py` runs repeatable local benchmarks:

1. target health check
2. `ai-asm scan`
3. DB endpoint/static-candidate summary
4. coverage calculation
5. agent trace analysis
6. `result.json` and `summary.md` output

## Usage

```bash
uv run python bench/run_bench.py --target crapi
uv run python bench/run_bench.py --target juice_shop
uv run python bench/run_bench.py --all
```

Results are written under `bench/results/<target>-<timestamp>/` and ignored by git.

## Local Candidate Services

`bench/services/compose.candidates.yaml` can start additional vulnerable apps for scanner diversity:

```bash
docker compose -f bench/services/compose.candidates.yaml up -d
docker compose -f bench/services/compose.candidates.yaml down
```

Candidate targets:

| target | stack | URL |
|---|---|---|
| `webgoat` | Java/Spring Boot | `http://localhost:8081/WebGoat/` |
| `dvwa` | PHP/Apache/MariaDB | `http://localhost:4280/` |
| `dvga` | Python/Flask/GraphQL | `http://localhost:5013/` |
| `vampi` | Python/Flask/OpenAPI | `http://localhost:5002/ui/` |
| `petstore` | Java/OpenAPI | `http://localhost:5014/` |
| `restful_booker` | Node/Express API docs | `http://localhost:5016/` |
| `go_httpbin` | Go HTTP/form service | `http://localhost:5015/` |
| `dvna` | Node.js/Express | `http://localhost:9092/` |
| `webgoat_dotnet` | ASP.NET container | `http://localhost:9000/` |

Run one candidate:

```bash
uv run python bench/run_bench.py --target dvga
```

`vampi` and `petstore` use documented path-list coverage fixtures from
OpenAPI-backed route tables. `restful_booker` and `go_httpbin` intentionally
keep path-list coverage enabled even though current discovery is weak there;
they track the prefixless API-doc limitation. Other candidates use `coverage:
none`; they are service-diversity smoke benchmarks until stable expected
endpoint lists are added.
`dvna` is registered but disabled for `--all` because it is mostly an
authenticated MPA with only one JSON-like API route, so it is not a useful API
surface coverage benchmark. `webgoat_dotnet` is also disabled for `--all`
because the old Mono-based image crashes on local Apple Silicon Docker; it can
still be run explicitly on a compatible Docker host with `--target
webgoat_dotnet`.

## Current Targets

Targets are configured in `bench/targets.yaml`.

| target | prerequisite | coverage |
|---|---|---|
| `juice_shop` | `juice-shop` on `localhost:3000`, `auth.json` from `ai-asm login` | public Juice Shop API path list |
| `crapi` | crAPI on `localhost:8888`, local OpenAPI spec at `.research/crapi/crapi-openapi-spec-running.json` | OpenAPI path coverage |
| `vampi` | VAmPI on `localhost:5002`, started via `bench/services/compose.candidates.yaml` | documented VAmPI API path list |
| `petstore` | Swagger Petstore on `localhost:5014`, started via `bench/services/compose.candidates.yaml` | documented Petstore API path list |
| `restful_booker` | Restful Booker on `localhost:5016`, started via `bench/services/compose.candidates.yaml` | prefixless API docs limitation benchmark |
| `go_httpbin` | go-httpbin on `localhost:5015`, started via `bench/services/compose.candidates.yaml` | prefixless HTTP docs/forms limitation benchmark |
| other candidates | started via `bench/services/compose.candidates.yaml` | none yet |

`juice_shop` intentionally requires `auth.json` because the v0.1 benchmark measures authenticated SPA traversal. Create it with:

```bash
uv run ai-asm login http://localhost:3000 -o auth.json
```

## Output

Each run directory contains:

- `scan.log`: raw command output
- `scan.db`: SQLite result DB
- `captures/`: scan artifacts, trace, and automatic exports
- `captures/api.yaml`: OpenAPI 3.0 export generated by scan
- `captures/flagged.yaml`: flagged items for manual review
- `captures/flagged.sh`: curl replay helper for flagged items
- `result.json`: machine-readable benchmark result
- `summary.md`: human-readable summary

`bench/results/latest.md` contains the last run summary for all selected targets.

## Thresholds

Each target can define:

- `coverage_min`
- `max_input_tokens`

By default threshold failures are reported but do not change the process exit code. Use:

```bash
uv run python bench/run_bench.py --all --fail-on-threshold
```

to fail the process when thresholds fail.
